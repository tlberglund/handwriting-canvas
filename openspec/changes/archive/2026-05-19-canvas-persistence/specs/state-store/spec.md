## ADDED Requirements

### Requirement: Object IDs are UUIDs
The `addObject` action SHALL assign each new `TextObject` an ID generated by `crypto.randomUUID()`. IDs SHALL be unique across sessions without requiring a persistent counter.

#### Scenario: Two objects created in separate sessions have distinct IDs
- **WHEN** a text object is added in one session and another is added after a page reload
- **THEN** both objects have distinct IDs with no collision

#### Scenario: No sequential ID counter exists in the store
- **WHEN** the store module is loaded
- **THEN** there is no module-level `nextId` counter; IDs are generated at call time via `crypto.randomUUID()`

### Requirement: Store persists a subset of state to localStorage
The store SHALL be wrapped with Zustand's `persist` middleware. The `partialize` option SHALL restrict the persisted payload to `textObjects` and `canvasBackground` only. The localStorage key SHALL be `'handwritten-title-canvas'`.

#### Scenario: Only the specified fields are written to localStorage
- **WHEN** the store updates for any reason
- **THEN** the value at `localStorage.getItem('handwritten-title-canvas')` contains only `textObjects` and `canvasBackground`

#### Scenario: Store hydrates on page load
- **WHEN** the page is loaded and a prior session's data exists in localStorage
- **THEN** `useStore.getState().textObjects` reflects the persisted array before the first render
